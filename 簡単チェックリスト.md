# ✅ 403エラー解決チェックリスト

## 🎯 5分でできる確認手順

### ステップ1: セキュリティプラグインを確認（最重要）

1. WordPress管理画面にログイン
   ```
   https://freeeeeeestyle.com/wp-admin/
   ```

2. 左メニュー「**プラグイン**」→「**インストール済みプラグイン**」をクリック

3. 以下のプラグインがあるか確認：
   - [ ] Wordfence Security
   - [ ] iThemes Security
   - [ ] All In One WP Security
   - [ ] Sucuri Security
   - [ ] Jetpack（セキュリティ機能）
   - [ ] その他のセキュリティプラグイン

4. **もしあれば**：
   - そのプラグインを**一時的に無効化**
   - 「無効化」をクリック

5. GitHub Actionsを再実行
   - 成功したら、そのプラグインが原因！

---

### ステップ2: パーマリンク設定を更新

1. WordPress管理画面で「**設定**」→「**パーマリンク設定**」

2. **何も変更せず**「**変更を保存**」をクリック

3. GitHub Actionsを再実行

---

### ステップ3: レンタルサーバーのWAFを確認

#### あなたのレンタルサーバーはどこですか？

<details>
<summary>📦 エックスサーバー</summary>

1. サーバーパネルにログイン
2. 「**WAF設定**」をクリック
3. 対象ドメインを「**OFF**」にする
4. 5-10分待つ
5. GitHub Actionsを再実行

</details>

<details>
<summary>📦 ロリポップ</summary>

1. ユーザー専用ページにログイン
2. 「**セキュリティ**」→「**WAF設定**」
3. 対象ドメインを「**無効**」にする
4. 5-10分待つ
5. GitHub Actionsを再実行

</details>

<details>
<summary>📦 ConoHa WING</summary>

1. コントロールパネルにログイン
2. 「**サイト管理**」→「**サイトセキュリティ**」
3. 「**WAF**」を「**OFF**」にする
4. 5-10分待つ
5. GitHub Actionsを再実行

</details>

<details>
<summary>📦 さくらインターネット</summary>

1. サーバコントロールパネルにログイン
2. 「**Webサイト/データ**」→「**WAF**」
3. 対象ドメインを「**無効**」にする
4. 5-10分待つ
5. GitHub Actionsを再実行

</details>

---

## 🎉 解決したら

### セキュリティプラグインが原因だった場合

プラグインを再度有効化して、設定を調整：

1. プラグインを有効化
2. プラグインの設定を開く
3. 「**REST API**」に関する設定を探す
4. 「**認証済みユーザーのアクセスを許可**」を有効にする

### WAFが原因だった場合

WAFを再度有効化して、REST APIのパスを除外：

1. WAFを「ON」にする
2. 除外設定で `/wp-json/` を追加（サーバーによって方法が異なる）

---

## 📊 どれが原因だった？

解決したら、以下のどれが原因だったか教えてください：

- [ ] セキュリティプラグイン
- [ ] レンタルサーバーのWAF
- [ ] パーマリンク設定
- [ ] その他

---

## 🆘 まだ解決しない場合

GitHub Actionsのログで以下を確認：

```
DEBUG: パスワードの先頭4文字: ****
エラーコード: ****
エラーメッセージ: ****
```

この情報を教えてください。より詳しく調査します。
